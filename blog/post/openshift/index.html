Although OKD or Openshigt projects has such a very good documentation, I’ve got in some troubles on my first setup attempts so I’ve decided to share some information to help newcomers to install Openshift.

You may run all the ansible setup through an external control machine or directly on your master node. Please ensure you save your copy of ansible inventory for future needs.

  1. Everything begins with a correct setup of your inventory (inventory/hosts.ini)

1.1. Add your masters both on masters and etcd sections. If you have dedicated etcd nodes please add it in the correct section.

[masters]
okd-masters-0[1:3].example.com

[etcd]
okd-masters-0[1:3].example.com

1.2. For a production environment you should setup at least two infra nodes. #FIXME: Add documentation references about the minimal number of infra nodes

[infra]
okd-infra-0[1:3].example.com

1.3. You should add both your masters, etcd and infra nodes together with your compute nodes. You shall also define each host role with openshift_node_group_name

[nodes]
okd-masters-0[1:3].example.com openshift_node_group_name="node-config-master"
okd-infra-0[1:3].example.com openshift_node_group_name="node-config-infra"
okd-compute-0[1:3].example.com openshift_node_group_name="node-config-compute"

1.4. If you are using an host with HA Proxy as the cluster Load Balancer (not recommended for Production environments) you need to add a lb section

[lb]
okd-lb-01.example.com

#FIXME: tell about lb section

1.5. All your settings goes on OSEv3:vars sections but you need to add some node groups before

[OSEv3:children]
masters
nodes
etcd

1.6.

[OSEv3:vars]

Deployment type: origin for OKD or XXX for Openshift

openshift_deployment_type=origin

Use Firewalld or IPTables otherwise (if False)

os_firewall_use_firewalld=True

Defines HA method when deploying multiple masters. Nowadays you should use Native

openshift_master_cluster_method=native

Defines internal cluster hostname

openshift_master_cluster_hostname=console-internal.example.com

Defines public cluster hostname

openshift_master_cluster_public_hostname=console.example.com

Define default subdomain for routed applications

openshift_master_default_subdomain=apps.example.com

Networking: subnets should be private and must not conflict with any existing blocks in your infrastructure to which pods, nodes or the master may require access

Pods network block

osm_cluster_network_cidr=10.128.0.0/14

Size of the per host subnet allocated for pod IPs. Defalt to 9 (subnets with 23 bits netmask)

osm_host_subnet_length=9

Services network block,

openshift_portal_net=10.90.0.0/16

Master API Port

openshift_master_api_port=443

Master console Port

openshift_master_console_port=443

Disable some checks

openshift_disable_check=disk_availability,memory_availability

Ansible user if not root

ansible_user=myuser

Become if not root

ansible_become=yes

Enable ELK Stack for logging

openshift_logging_install_logging=true openshift_logging_es_pvc_dynamic=true openshift_logging_es_pvc_size=100Gi openshift_logging_es_cluster_size=3 openshift_logging_es_number_of_replicas=1

Enable Metrics

openshift_metrics_install_metrics=true openshift_metrics_storage_volume_size=20Gi openshift_metrics_cassandra_storage_type=dynamic openshift_metrics_duration=7 openshift_metrics_resolution=30s


2. Setup your master nodes with dedicated storage to Docker with LVM.
If docker is already initialized please stop it and clean up /var/lib/docker

$ echo “STOREAGE_DRIVER=overlay2 DEVS=/dev/sdb VG=docker-vg” | sudo tee /etc/sysconfig/docker-storage-setup

$ sudo docker-storage-setup


3. Use fully qualified names for your hosts

4. Meet the required hardware or tell setup to ignore checks

5. Run prerequisites.yml playbook

$ ansible-playbook -i inventory/hosts.ini playbooks/prerequisites.yml


6. Run deploy_cluster.yml playbook

$ ansible-playbook -i inventory/hosts.ini playbooks/deploy_cluster.yml


If the deploy fails you may check the message for reaston it 'd failed and rerun this task. For instance you may comment already executed playbooks.

#- import_playbook: openshift-checks/private/install.yml # #- import_playbook: openshift-node/private/bootstrap.yml ```

  1. Common errors during deploy

7.1. If you have any network communication problem between nodes, the Bootstrap Step mail fail. Please be sure each node can access all the other nodes of the cluster.

7.2. If Node Join step fail, you may haven’t configured your nodes with their fully qualified names. Please use hostnamectl set-hostname hostname.example.com to setup your node. You may run hostname to check if it returns just your hostname or hostname.example.com.

7.3. In case you are deploying an HA setup, you must configure your LB with the Master nodes just before you start deploy or it will fail.

7.4. Your apps endpoint needes a separated LB as well
